# VFX Slate Maker

A cinematic VFX slate generator that renders **HTML/CSS** slates to **OpenEXR** image sequences. Built with **PySide6**, **Qt WebEngine** (live preview + offscreen capture), and **OpenImageIO**.

### Pipelines and automation (no Nuke license)

Facilities often author slates in **Nuke** (or similar compositors). That is a good fit for artists at the desk, but running Nuke on a farm or in **headless automation** to generate slates still pulls a **Nuke license**—per machine or per job—just to burn in metadata frames. For high-volume or purely mechanical slate generation, that license cost is easy to avoid.

Slate Maker outputs **OpenEXR** from **HTML/CSS** using Qt and OpenImageIO only. You can script it, run it on generic render boxes, or use a **bundled binary** in CI so slate passes do **not** consume a Nuke (or other comp package) license, while Nuke stays reserved for actual compositing.

## Features

- **Live preview** — Form edits push JSON into the page; the template exposes `updateSlate(data)` for redraws
- **Qt WebChannel** — Optional bridge for JS logging / future hooks
- **EXR output** — Numbered frames, RGBA, half or float, sRGB or linear with transfer-aware conversion
- **Presets** — Common delivery resolutions and frame rates
- **Dark UI** — Fusion style tuned for long sessions

## Requirements

- **Python 3.11**
- [uv](https://docs.astral.sh/uv/)

```bash
cd apps/slate_maker
uv sync
```

`oiio-python` pulls native wheels on most platforms; on macOS, if install fails for your arch, a system **OpenImageIO** via Homebrew may be required.

Qt WebEngine ships with the PySide6 stack used by this app; if you see import errors for `QtWebEngineWidgets`, ensure your environment matches the PySide6 version in `pyproject.toml`.

## Usage

```bash
uv run python main.py
```

(`pyproject.toml` also declares a `slate-maker` console script for environments that install entry points on the path; with uv in this repo, `python main.py` is the reliable invocation.)

1. Fill shot metadata (project, sequence, shot, version, artist, etc.).
2. Set frame range, FPS, resolution, bit depth, colorspace, and **output directory**.
3. Use the preview (middle-mouse pan, scroll zoom, **F** to fit).
4. **Render EXR Sequence** writes frames.

Output files: `{shot}_slate.{frame:04d}.exr` in the chosen directory.

### From the monorepo root

With the [workspace `pyproject.toml`](../../pyproject.toml), after `uv sync` at the repo root:

```bash
uv run --project apps/slate_maker python apps/slate_maker/main.py
```

## Customizing the slate

- **Template:** [`templates/slate.html`](templates/slate.html) — layout and styling; Tailwind build input is [`templates/input.css`](templates/input.css).
- **Compiled CSS:** After changing classes, rebuild CSS:

  ```bash
  make tailwind
  ```

  Requires the [Tailwind CSS v3 CLI](https://tailwindcss.com/docs/installation) on your `PATH` (or set `TAILWIND=/path/to/tailwindcss`).

- **Behavior:** The preview calls `updateSlate(...)` with the form’s JSON; keep that contract when editing the template.

## Project layout

```
apps/slate_maker/
├── main.py              # Main window, preview, WebChannel bridge
├── pyproject.toml
├── resources.qrc        # Qt resources (icons)
├── public/              # Icons for .qrc / bundling
├── templates/
│   ├── slate.html       # Slate markup + Tailwind classes
│   ├── input.css        # Tailwind source
│   ├── tailwind.css     # Generated (run make tailwind)
│   └── tailwind.config.js
└── src/
    ├── constants.py     # Resolutions, FPS, bit depth, colorspace presets
    ├── widgets.py       # Form panel
    ├── worker.py        # WebEngine capture → EXR
    └── rc_resources.py  # Generated by make resources
```
